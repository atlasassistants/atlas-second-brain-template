# Strategy — Sample Exec (Sarah Chen / Gridline)

> **This is a fictional exec. Read these files as a demonstration of shape, not a target to mimic.** The strategy here shows what `/content-strategy-onboard` produces when run for a real exec. Sarah's domains, pillars, cadence, and commercial mapping are specific to her. Your exec's population will look different.

This directory contains Sarah Chen's complete content strategy — the 7 files that define what she writes about, who reads it, when it ships, where it goes, and how the voice frame adjusts per domain.

---

## What this is

A worked example of the writing-system's strategy layer, populated for a fictional B2B SaaS founder with:
- 3 content domains (one per audience need, one cross-audience)
- 9 pillars (3 per domain)
- A bi-weekly newsletter rhythm (~24 long-form pieces/year)
- One commercial product (Gridline) mapped directly to one domain and indirectly to the others

**Read the strategy layer before reading any drafts.** The domain + pillar context makes the draft choices legible.

---

## Files in this directory

| File | What it covers |
|---|---|
| [[domains]] | Three content domains — Engineering Velocity at Scale, AI-Native Infrastructure, Founder-as-Builder. Identity, audience, commercial rationale, and boundary notes for each. |
| [[pillars]] | Nine pillars (3 per domain) with canonical kebab-case slugs. These slugs are used verbatim in draft frontmatter and skill invocations. |
| [[audience-mapping]] | Domain-to-audience matrix. D1 and D2 serve SaaS founders primarily; D3 is Sarah's dual-audience domain (founders + IC-to-leadership transitioners). Pairing rules for when to publish companion pieces. |
| [[commercial-mapping]] | Direct CTA vs. soft pointer vs. brand-only per domain. D2 is direct (Gridline demo); D1 is soft pointer; D3 is personal brand authority with no direct CTA. |
| [[cadence]] | Per-domain publishing targets: D1 (10/year), D2 (8/year), D3 (6/year) = 24 long-form pieces/year on a bi-weekly newsletter cycle. Per-pillar volume targets included. |
| [[distribution]] | Channel matrix and repurposing rules per domain. D1 and D2 run full chains (newsletter + LinkedIn + X); D3 is newsletter + LinkedIn only. D2 is the only domain with gated content potential. |
| [[voice-frame]] | Per-domain voice layer — which anchor moves lean heaviest, which moves to avoid, and domain-specific kill-list additions on top of the global [[../kill-list]]. |

## How to read these files

Start with [[domains]] — it establishes the three-domain structure and why each domain exists for Sarah specifically. Then read [[pillars]] to see the 9 sub-topics and their canonical slugs. The remaining files each add a layer:

- **[[audience-mapping]]** — answers *who reads this?*
- **[[commercial-mapping]]** — answers *what does this convert toward?*
- **[[cadence]]** — answers *how often does this ship?*
- **[[distribution]]** — answers *where does this land?*
- **[[voice-frame]]** — answers *how does the voice shift per domain?*

After reading the strategy, open [[../drafts/sample-newsletter]] to see the strategy applied to an actual draft.

## What this demonstrates

- A 3-domain structure (not all execs need 4 or more)
- A cross-audience domain (D3 serves both Sarah's audiences simultaneously — this is specific to her profile)
- Commercial mapping where one domain is direct-to-product and two others are indirect — a typical founder/SaaS pattern
- The distinction between a scheduled domain (D1, D2) and an opportunistic one (D3) — cadence realism, not aspiration
- Domain-specific voice adjustments that stay true to the same underlying voice signature
