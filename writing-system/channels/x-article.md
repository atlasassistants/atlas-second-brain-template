# Channel: X long-form article

> **Status:** v1. Format rules for long-form articles posted on X (formerly Twitter). Same Garry-Tan skeleton as newsletters with X-native tweaks.

---

## X article skeleton

Same fundamental structure as `channels/newsletter.md`, with X-specific delivery adjustments.

1. **Title.** X long-form articles have a title field. Specific over clever. Avoid clickbait — {{VAULT_OWNER_SHORT_NAME}}'s audience tunes that out.
2. **Cover image.** *(optional)* If used, a real screenshot or simple text-on-image. Avoid stock graphics.
3. **Opening hook (confessional).** 1–3 paragraphs. Same pattern as newsletter open: drop into a specific moment. Named subjects, real numbers. The first 280 characters often appear as a preview, so the hook needs to land in that space.
4. **The pivot.** One paragraph naming what the moment revealed.
5. **Tactical middle.** Same as newsletter: receipts, sub-headings, sub-section breaks. Inline images / screenshots / diagrams allowed and encouraged for X.
6. **Section-break pullbacks.** 2–4 sentence philosophical reframes at major breaks. Same rule as newsletter: vary, don't repeat.
7. **Philosophical close.** 1–3 paragraphs reframing what the tactic means.
8. **Closing CTA / sign-off.** Different from newsletter:
   - **No P.S. / P.P.S.** (those are email artifacts).
   - Optional: a short final sentence pointing to a related thread, a newsletter signup, or a community.
   - No *"Your Pal, {{VAULT_OWNER_SHORT_NAME}}"* sign-off (X articles are signed by author handle automatically).

## What changes from newsletter

| Element | Newsletter | X article |
|---|---|---|
| Sign-off | *Your Pal, / {{VAULT_OWNER_SHORT_NAME}}* | None (handle is the signature) |
| P.S. | Direct reply ask | None |
| P.P.S. | Optional tactical add-on | None |
| Distribution | Email inbox (subject line is the hook) | X feed (cover image + first 280 chars are the hook) |
| CTA | Direct reply | Reply / RT / save / link to subscribe to newsletter |
| Visual elements | Minimal images, screenshots | More visual — inline screenshots, diagrams, charts encouraged |
| Length | 800–1500 words typical | 1000–2500 words typical (X long-form sweet spot) |

---

## X article-specific format rules

- The first 280 characters appear as the preview tweet when shared. Make those characters do work — the hook has to land in that space.
- Use inline images / screenshots more freely than in newsletters. The medium rewards visual proof.
- Heading hierarchy is shallower than newsletter (X article rendering favors fewer H2s, more flowing prose).
- No email-style P.S. or sign-off.
- End with a single specific link if used (newsletter subscribe, related thread, GitHub repo for a referenced tool, etc.).

## When to write an X article vs. newsletter

| Situation | Use newsletter | Use X article |
|---|---|---|
| Want to reach the existing list | ✅ | |
| Want to build new audience on X | | ✅ |
| Topic is intimate / behind-the-scenes | ✅ | |
| Topic is provocative / contrarian (designed to spread) | ⚠️ | ✅ |
| Includes screenshots / visual proof | ⚠️ | ✅ |
| Want direct reply for relationship building | ✅ | |

**Common pattern:** Same idea written as both. Newsletter version goes to the list with the philosophical pullback at full length. X article version is similar but optimized for the feed and link-shareable.

If both are written from the same source idea, the `repurposing` skill is NOT what runs (that's for shorter formats). Both are long-form variants — write each fresh from the source idea using `long-form-writing` skill, with the channel file matching the surface.

## Examples: in-voice vs. out-of-voice

### Title
- **OUT:** *"5 Productivity Tips Every Founder Should Know"*
- **IN:** *"why my friend pre-sold a tech stack he hasn't built yet"*

### Opening (the first 280 chars carry weight on X)
- **OUT:** *"In this article, I'll share my insights on how AI is transforming the way founders work and what you can learn from this shift…"*
- **IN:** *"A friend told me 'my life has changed' two weeks after I taught him to build with AI. He's in healthcare. Two days of teaching, four days of building, and he replaced four months of his team's policy work."*
